# Part 3: Layout Decisions

## Key choices

- The hero panel sits above the app so the project reads like a complete dashboard instead of a single card.
- The search area stays near the top so it is easy to reach on all screen sizes.
- The current weather section and forecast section are visually separated for a clearer information hierarchy.
- The forecast becomes a 5-column grid on desktop because that is the easiest way to compare multiple days quickly.

## Usability impact

- Buttons remain large enough for touch interaction
- The app avoids horizontal overflow
- Labels and helper text stay visible even on narrow screens

## Reviewer note

The layout was designed to look intentional, but the main goal was still clarity: the checker should immediately understand where to search, where to read the current weather, and where to inspect the forecast.
